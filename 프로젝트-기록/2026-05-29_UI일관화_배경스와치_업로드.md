# 2026-05-29 — UI 일관화: 배경 사진 스와치 + 내 사진 업로드 + 컨트롤 통일

## 배경
사용자 지적: "배경이 '침실/쿠시노' **글자뿐**이라 뭔지 모른다 — 썸네일로 보이거나, 내 사진을 넣게 해달라. 드롭다운 등 다양한 UI로 일관되고 세련되게."
+ "진행할 때 **폰트 위계질서**와 **문구**(개발자 용어/불친절/혼란)도 매번 점검."
확정 범위(질문): 핵심+일관화 / 세그먼트는 보이게 유지·긴 목록만 드롭다운 / 업로드는 배경+제품 둘 다. 계획: `.claude/plans/ui-atomic-pelican.md`.

## ✅ 오늘 한 것

### 1) 배경 사진 = 썸네일 스와치 + "내 사진 추가" (헤드라인)
- 텍스트 버튼 3개 → **썸네일 스와치 그리드**(`.swatch-grid`): 없음(대각선 모티프) / 침실·쿠시노(실제 사진 + 캡션) / 내 사진(올린 경우) / **내 사진 추가**(점선+＋).
- `renderBgGroup()`·`bgActiveId()`(data URL이면 'custom')·`applyBgChoice()`('custom' 유지) 갱신. `data-bg` 값(off/bedroom/cushino)은 **그대로 유지** → verify-bg 무영향.

### 2) 내 사진 업로드 (배경 + 제품 공용)
- 신규 `downscaleToDataUrl(file,{maxDim,quality})` — 업로드 사진을 **캔버스로 축소 → data URL**로 `state`에 박음. 서버 엔드포인트 추가 없이 **미리보기·인쇄 내보내기 공통** 반영(전체 state가 `/api/export`로 POST → 헤드리스 캡처까지 도달).
- `handleBgUpload`(maxDim 2400 ≈ 인쇄 100DPI) / `handleProdUpload`(maxDim 1200, 제품 카드는 작음). 너무 작은 사진은 기존 해상도 안내가 자동으로 잡음. 비이미지·실패는 친절 alert.
- 제품 피커(`openImgPicker`) 맨 위 "내 사진" 그룹 + "카드에 맞춰 잘려 보일 수 있어요" 안내. `#bgUpload`/`#prodUpload` 숨은 input + boot 바인딩.

### 3) 컨트롤 어휘 통일 (드롭다운은 긴 목록만)
- 톤·결제 모드 = **세그먼트 버튼 유지**(3~4개는 한눈에·즉시 반영이 드롭다운보다 나음). `.tone-btn` 글자 13.5→**14px**로 위계 정리.
- 네이티브 `<select>`(재질·후기 색상/아이콘) → **커스텀 chevron(회색 SVG) + 동일 테두리/포커스**로 통일(`select.field-input { appearance:none }`, `appearance:auto` 강제 제거). 포커스 시 chevron 안 지워지게 `.field-input:focus` 를 `background-color`로.
- 신규 공용 `.swatch`/`.swatch-cap`/`.swatch--none|image|add` + `.field-sep`(다듬기 항목 구분선).

### 4) §0 가로지르는 점검 (사용자 강조)
- **타이포 위계**: 손댄 컨트롤 글자크기 사다리 안(12.5·14px), 12px 미만 0. 같은 역할 같은 스타일.
- **카피 친절도**: 추가 문구 전부 친근체·무개발자용어("JPG·PNG"·"잘려 보일 수 있어요"처럼 쉬운 말, data URL/DPI/case 노출 0). verify-ui가 자동 스캔.
