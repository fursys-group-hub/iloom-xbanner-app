- **시작 화면 배경**(`.start-screen`): 옛 흰 그라데이션 → 침실 배경 사진 + 옅은 흰 스크림(사용자 요청으로 진하게 .45→.28)으로. 흰 카드·글자 가독 유지.
- **브랜딩 — "일룸" 텍스트 → iloom 로고 이미지**: 시작 타이틀·사이드바 헤더의 "일룸 X배너 어플" 에서 "일룸" 을 `/assets/products/iloom-logo.png` 로 교체(`.brand-logo`, em 기준 크기). 시작 화면은 조금 더 크게(`.start-title .brand-logo` 0.92em). (매장명 placeholder·브라우저 탭 제목은 내용/메타라 제외)
- **배경 사진 = 가로 폭 맞춤(최종)**: 사용자 실제 의도는 `cover`(높이 기준 확대 → 배너 중간까지만 보임)가 아니라 "**가로 사이즈에 맞춰 배치**". banner.css 이미지 레이어 `cover` → `center bottom / 100% auto`(폭 100%·높이 비율·하단 정렬 → 전체 사진이 폭에 맞게 하단에 깔림). 위치/확대 **슬라이더는 시도했다가 "사용자친화적이지 않다"는 피드백으로 제거**. 검증 `verify-bgadjust`(슬라이더 없음 + cover 아님 + 폭100%) 4/4.
- **경계선 제거 — 사진을 `<img>`+마스크로 전환(최종)**: 배경별 비율이 제각각(침실=세로 윗변 50% / 나머지 가로 윗변 78%)이라 고정 그라데이션으론 경계를 못 가림. → 배경 사진을 `render-banner.js` 가 `.x-banner` 첫 자식 `<img class="banner-bg">`(폭 100%·하단 정렬)로 주입하고, CSS `mask-image: linear-gradient(to top,#000 38%,transparent 90%)` 로 **사진 윗부분을 페이드**(사진 높이 무관하게 경계 없음). `.has-bg` 배경은 크림만. 합성·순서는 `.x-banner{isolation:isolate}` + `.banner-bg{z-index:-1}`(콘텐츠는 그 위). `applyBackground` 는 has-bg 토글만(--bg-image 제거).
- **배경 위 iloom 로고 = 흰 글자만**(검은 박스 없이): 로고 PNG가 **흰 배경+검정 글자**(코너 255,255,255 / 가운데 55,46,41 — 그래서 일반 배너는 `mix-blend-mode:multiply`로 흰 배경을 크림에 녹임). 배경 사진 위에선 `.has-bg .footer img { filter: invert(1); mix-blend-mode: screen }` → invert 로 흰글자/검정배경 만든 뒤 **screen 으로 검정배경을 투명 처리** → 사진 위에 흰 iloom 만 남음. 합성되도록 `.x-banner { isolation: isolate }` + `.banner-bg { z-index:-1 }`(콘텐츠는 그 위, 별도 z 룰 불필요). ★ `brightness(0) invert(1)`(흰 블록)·`invert(1)` 단독(검은 박스)은 둘 다 안 됨 — screen 필수.
