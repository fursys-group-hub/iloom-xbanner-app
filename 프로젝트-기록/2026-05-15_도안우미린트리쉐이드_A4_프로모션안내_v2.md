### 4. 인쇄용 PNG 추출
- HTML 구조를 단톡방 QR 시안과 통일:
  - 클래스 이름: `.a4-page` → **`.a4-sheet`** (export-a4-png.mjs 호환)
  - 사이즈: `210mm × 297mm` → **`794px × 1123px`** (@96dpi 환산값)
  - padding: `14mm 16mm 12mm` → **`53px 60px 45px`** (비례 변환)
  - `@page { size: A4 portrait }` → `@page { size: A4 }`
- 스크립트 실행: `node scripts/export-a4-png.mjs "시제품/도안우미린트리쉐이드_A4_v2.html"`
- 결과: `시제품/도안우미린트리쉐이드_A4_v2_인쇄용.png` (**2382×3369px / 300dpi A4 / 2.4MB**)

## 🤔 결정한 것 (규칙서 강화 후보)

### A. A4 프로모션 안내 시안 — 단톡방 QR과 헤더 통일성
- **apt-name** : 45px / weight 700 / letter-spacing -0.05em / line-height 1.15
- **main-title** : 50px / weight 800 / coral / letter-spacing -0.05em / line-height 1.1
- 단톡방 QR (`apt-name` 45px/700, `channel-subtitle` 50px/800)과 동일 → **출력물 3종 세트 헤더 시각 통일**

### B. A4 시안 페이지 셋업 표준
- 클래스명 : `.a4-sheet` (export-a4-png.mjs 의 selector 와 일치 필수)
- 사이즈 : `794px × 1123px` (mm 대신 px — 스크립트 viewport와 일치)
- `@page { size: A4; margin: 0; }`
- body padding: `40px 0` (스크린 미리보기 여백)

### C. 배경 이미지 적용 — A4 시안에도 X배너 표준 그대로
- 패턴: `linear-gradient(to bottom, 크림100 0% → 크림100 50% → 크림85 65% → 크림55 82% → 크림25 100%) + url(...) center bottom / cover`
- 배경이 깔린 영역(하단)의 텍스트는 **흰색 + (옵션) text-shadow** 로 가독성 확보
- 그림자 추가하면 인쇄 시 너무 진해질 수 있음 — 일단 흰색만으로 시도, 안 보이면 그림자 추가

### D. A4 케이스별 변형 패턴 (X배너 v14 ↔ A4 v2 매핑)
| X배너 v14 영역 | A4 v2 변형 |

| 헤더 (WELCOME HOME + apt + main + period) | WELCOME HOME 제거, 나머지 유지 (폰트는 단톡방 QR과 통일) |
