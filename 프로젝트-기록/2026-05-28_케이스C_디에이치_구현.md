### ★ 기간 추출 버그 수정 (전 케이스 정확도 향상)
1. **패턴1 종료일 분해 버그**: "25년 10월 18일~19일" 에서 종료 "19일"이 `(\d{1,2})?월?(\d{1,2})일` 에 걸려 "1월 9일"로 쪼개짐 → 종료 월은 "월"이 붙을 때만 인정하도록 수정. 종료 월 생략 시 시작 월 상속(사용자 확인: "19일은 25년 10월 생략한 것").
2. **풀데이트 폴백**: 기간 라벨이 연도 없는 "7/31 - 8/2" 면 실패 → "박람회 현장 계약 혜택 : 2026/07/31(금)~08/02(일)" 풀데이트 줄을 폴백으로 파싱 (패턴0 신규: 4자리 연도 시작 + 월/일 끝 같은 해).
- 결과: 디에이치 `2026-07-31~08-02` / 대구범어 `2025-10-18~10-19` 교정. 한화/잠실/교대역 회귀 없음.

## 🔧 추가 — 제품 카드 직접 추출 (사용자 지적: "모션테이블/모션데스크가 왜 안 보여?")
초기엔 프리셋(옷장단독→키큰옷장, 모션→리빙모션베드)으로 2종만 나왔다. PDF에 제품 3종이 명확히 적혀 있어("키큰 옷장 2개 포함 150만원 이상 구매 시 10만 포인트 증정" / "리빙 모션베드 구매 시 10만 포인트 증정" / "업모션 테이블, 모션데스크 구매 시 5만 포인트 증정") 직접 추출하도록 `promotions.js extractProductPromos` 추가.
- 패턴: "<제품> ... 구매 시 N만 포인트 증정" → 제품명(키워드 매핑)/조건(키워드~"구매 시")/포인트(N만→N0,000)
- PRODUCT_DEFS: 키큰옷장→컬렉트 / 리빙모션베드 / 모션데스크·업모션→업모션 이미지
- 결과: 디에이치 자동 플로우 제품 카드 3종 전부 v10 정확 일치(조건·포인트 포함). to-app-state 는 추출 결과 우선, 없으면 키워드 프리셋 폴백.
