# 2026-05-08 — 품의서 자동 추출 로직 설계

> 9개 품의서 분석을 바탕으로 어플이 어떻게 데이터를 자동 추출할지 구체화한 설계 문서.
> 이 문서를 보면 개발자가 바로 코드를 짤 수 있도록 정리.

---

## 🎯 전체 흐름도

```
[1] 사용자가 품의서 PDF 업로드
       ↓
[2] PDF에서 텍스트 추출 (pdf.js)
       ↓
[3] "닻 라벨" 검색해서 데이터 위치 찾기
       ↓
[4] 라벨 옆/아래에서 값 추출 (정규식)
       ↓
[5] 데이터 정규화 (날짜 형식 통일, 약명 괄호 제거 등)
       ↓
[6] 케이스 분기 판단
   - 박람회 참여? 미참여?
   - 매장 1개? 2개? 3개?
   - 1아파트? 2아파트?
       ↓
[7] 미분류·실패 항목 사용자에게 안내
       ↓
[8] 사용자 확인/수정
       ↓
[9] 카피 자동 생성 + 시안 미리보기
```

### 1-4. 약명 (가장 중요한 식별자)

**닻 패턴:**
```
"입주아파트 약명 :" 다음 괄호
"아파트 약명 :" 다음 텍스트 또는 괄호
"수주건명 :" 다음 "(약명)" 추출
```

**9건 실제 형식:**
| 케이스 | 형식 | 약명 |
|---|---|---|
| 광주 | `* 입주아파트 약명 : (광주데시앙)` | 광주데시앙 |
| 창원 | `3) 아파트 약명 : (창원롯캐)` | 창원롯캐 |
| 디에이치 | `* 입주아파트 약명 : (디에이치)` | 디에이치 |
| 부산센텀 | `수주건명 기재 : 고객명(입주)(현장)(두산오션)` | 두산오션 |
| 수원성중흥 | `약명 : (수원성중흥)/ 추가혜택 : (수원2)` | 수원성중흥 |
| 잠실 | `* 입주아파트 약명 : (잠실래미안) / (잠실르엘)` | 잠실래미안 / 잠실르엘 |
| 대구범어 | `3) 아파트 약명 : 범어2차` | 범어2차 |
| 한화포레나 | `2) 아파트 약명 : (월평공원한화)` | 월평공원한화 |
| 교대역모아 | `3) 아파트 약명 : 교대역모아` | 교대역모아 |

**추출 정규식:**
```
약명\s*[:：]\s*[\(\(]?([^\)\)\s,\/]+)[\)\)]?
```
괄호가 있을 수도 없을 수도 있고, 슬래시(/)로 여러 약명이 나열될 수도 있음.

## 📊 Part 3 — 혜택표 컬럼 분기 규칙

### 3-1. 5단계 구간은 항상 동일
```
구매금액대: 200만원 / 300만원 / 500만원 / 700만원 / 1,000만원 (또는 1000만원)
```
9/9 케이스 동일. 구간 인식만 잘하면 됨.

### 3-2. 컬럼 구조 — 3가지 변형 자동 인식

#### 변형 A: **4컬럼 (구버전)** — 25년 작성
```
| 구매금액대 | 기존혜택(상시) | 추가혜택(박람회전용) | 총 혜택금액 |
```
- 사례: 부산센텀, 대구범어, 창원
- 인식 키워드: `추가혜택(박람회전용)` 또는 `추가혜택`

#### 변형 B: **5컬럼 (신버전)** — 26년 작성
```
| 구매금액대 | 기존혜택(상시입주) | 입주 아파트 현장 계약 혜택 | LG 연계혜택 | 총 혜택 |
```
- 사례: 광주데시앙, 디에이치 방배, 한화포레나
- 인식 키워드: `현장 계약 혜택` + `LG 연계혜택`

#### 변형 C: **6컬럼 (잠실 특수)**
```
| 구매금액대 | 아파트특별혜택 | 기존혜택(상시) | LG연계혜택 | 박람회현장 혜택 | 현장계약 최대혜택 |
```
- 사례: 잠실래미안 + 잠실르엘
- 인식 키워드: `아파트특별혜택` + `현장계약 최대혜택`

### 3-3. 자동 분기 의사코드 (Pseudocode)
```python
def detect_table_type(table_headers):
    if "아파트특별혜택" in headers and "현장계약 최대혜택" in headers:
        return "6컬럼_잠실형"
    elif "현장 계약 혜택" in headers and "LG 연계" in headers:

```

## 🔀 Part 4 — 케이스별 분기 로직

### 4-1. 박람회 참여 vs 미참여

```python
def detect_expo_participation(text):
    if "박람회 미참여" in text or "박람회미참여" in text:
        return "미참여"
    elif "박람회" in text and "장소" in text:
        return "참여"
    else:
        return "확인필요"
```

| 케이스 | X배너 출력 | A4 POP | 리플렛 | 동작 |
|---|---|---|---|---|
| 참여 (대부분) | O | O | 선택 | 표준 모드 |
| 미참여 (수원성중흥) | X | O | **필수** | 매장 단독 모드 |

### 4-2. 매장 구성 인식

**감지 패턴:**
```
"공략 매장" 다음에 / 또는 , 로 구분된 매장 개수
```

| 매장 수 | 케이스 | 처리 |
|---|---|---|
| 1개 | 창원, 논현, 수성 | 단독 매장 시안 |
| 2개 | 광주, 한화, 교대 | 매장 박스 2개 |
| 3개 | 부산센텀+연제+동부산 | 매장 박스 3개 |

**매장 코드 자동 부여:**
```python
PREMIUM_SHOPS = ["분당서현", "송파", "강동아이파크", "논현", "부산센텀"]
def assign_shop_code(shop_name):
    if shop_name in PREMIUM_SHOPS:
        return "(P)"
    else:
        return "(B)"
```

> ※ 직영사업팀 표기 발견 시 → 유통경쟁력강화팀으로 자동 변환
> ※ (투자B) 발견 시 → (B)로 자동 통합

### 4-3. 1아파트 vs 2아파트 동시 공략

**감지 패턴:**
```
제목에 "&" 또는 "+" 가 있고 약명이 2개 이상 추출되는 경우
예: "잠실래미안 입주박람회 참가 & 잠실르엘 공략의 건"
```

| 케이스 | 처리 |
|---|---|
| 1아파트 | 시안 1세트 (X배너 + A4) |
| 2아파트 | 시안 2세트 (각 아파트별 X배너 + A4) |

## 📝 Part 5 — 자유 텍스트 (사용자 확인 영역)

자동 추출이 어렵거나 정확도가 낮은 영역. 어플은 추출 시도 후 **사용자에게 "맞나요?" 확인 받기**.

### 5-1. 헤드카피 후보 자동 생성

**자동 계산 가능한 것:**
- 최대 혜택 = 1000만원 구간 총혜택 (표에서 자동 추출)
- 박람회 일수 = 박람회 종료일 - 시작일 + 1
- 매장 수 = 공략 매장 개수

**자동 카피 후보 (어플이 3개 제안):**
```
[A] 금액 강조: "최대 {최대혜택}만원 특별 혜택, 지금 바로 만나 보세요!"
[B] 기간 강조: "{일수}일간의 특별한 혜택을 만나보세요"
[C] 축하 톤: "{아파트명} 입주를 진심으로 축하드립니다"
```

→ 사용자가 1개 선택하거나 직접 수정

### 5-2. 후기 이벤트 종류 분류

키워드 매칭 후 사용자 확인:
```
"네이버 예약" → 네이버 예약 후기
"입주카페", "입주 카페" → 입주카페 후기
"SNS", "블로그", "인스타" → SNS 후기
"단톡방" → 단톡방 후기
"LSA 포상", "LSA 적극성" → LSA 우수후기 포상
```

### 5-3. 단톡방 경품 월별 표 추출

## 🛠️ Part 7 — 구현 시 주의사항

### 7-1. PDF 텍스트 추출의 함정
- PDF 표는 텍스트 순서가 예측 불가능 → **위치 좌표 기반 보정** 필요
- pdf.js 의 `getTextContent()` 결과를 좌표(x, y)로 정렬

### 7-2. 한글 처리
- 정규식에서 `\w` 는 한글 매칭 안 됨 → `[가-힣A-Za-z0-9]` 사용
- 공백 변형: 일반 공백 / 비파괴 공백(NBSP) 모두 처리

### 7-3. 숫자 파싱
- "1,690세대", "180,000,000원", "1.8억" 모두 다른 형식
- 통일 변환: 콤마 제거, "억" → 100,000,000 곱하기

### 7-4. 날짜 파싱
- "2026년 4월" / "26년 4월" / "2026/04" / "4월" 등 다양
- 정규식 + 컨텍스트 (앞뒤 키워드)로 보정

### 7-5. 동의어 사전 확장
- 새 작성자 등장하여 새 표현 발견 → 사전에 추가
- 사전이 코드와 분리(JSON) → 비개발자도 수정 가능하게
