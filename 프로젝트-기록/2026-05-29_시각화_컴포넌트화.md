# 2026-05-29 — 편집화면 시각화 강화 + 컴포넌트화 (아이콘·체크·모달 상태·상태 표시)

## 배경
디자인 시스템 정리(흰 카드 + 검정/빨강/코랄/그린) 후, 사용자가 "레퍼런스 모달 키트처럼 아이콘·체크·상태 표시를 더 쓰고 반복 UI를 컴포넌트로 통일하자" 요청 → 4개 방향 전부 선택.
기존: 신뢰도=색 점, "확인"=텍스트, 토스트=색 점, 아이콘은 `PLUS_SVG` 하나, 정보 목록 3개(요약·발주·신뢰도) 따로.
제약: **인라인 SVG 아이콘(라인, currentColor) — 이모지 금지**. 색은 의미 토큰, 배너 불변.

## ✅ 한 것

### 1. 아이콘 헬퍼 (토대)
- `app.js`에 `ICON` 맵(check·alert·xCircle·dash·trash·download·copy·close·info·spinner) — 인라인 SVG, `stroke=currentColor`.
- `editor.css` `.icon`(1em, currentColor) + `@keyframes icon-spin`/`.icon-spin`.

### 2. 아이콘·체크 시각화
- **신뢰도 칩** `confChip()`: 색 점 → 아이콘(정확=`check` 그린·확인권장/필요=`alert` 코랄·못찾음=`dash` 중립). 색은 `.conf-*`에서 상속. (textContent 라벨 유지 → verify-trust 무영향)
- **"확인" 버튼** `.rv-item-ok`: 앞에 `check` 아이콘.
- **버튼 아이콘 주입**(boot): PNG/PDF/.txt=`download`, 발주서 복사=`copy`, 닫기(편집/사진 팝오버)=`close`, 표 행 삭제 `.te-del`=`trash`.
- **토스트** `showToast()`: 색 점 → 아이콘(ok=`check` 그린·err=`xCircle` 빨강·busy=`spinner`).

### 3. 모달 상태 아이콘
- `confirmDialog()`에 `variant`/`icon` 추가 → 제목 위 `.ui-dialog-icon`(틴트 원 + 아이콘). danger=빨강 `alert`·warning=코랄 `alert`·success=그린 `check`·info=중립.
- 새 배너/기본값 복원=danger(빨강), 다운로드 안전확인=warning(코랄).
- 다운로드 완료는 블로킹 모달 대신 **토스트 그린 체크**로(매번 모달은 거슬림 — 트레이드오프).

### 4. 컴포넌트 통일
- **spec-list 컴포넌트**: `editor.css` `.spec-list`/`.spec-row`(+`.spec-k`/`.spec-v`/`.is-bad`)로 통일. `renderDoneSummary`·`renderOrderAuto`가 `spec-row`(기존 `done-row`/`order-row` 클래스도 함께 부여 → 테스트·선택자 보존) 사용. 중복 `.done-row`/`.order-row` 규칙 제거. 행 크기·테두리 단일 소스.
- 세그먼트·배지는 이미 일관 패턴이라 별도 리팩터 보류(필요 시 후속).

### 5. 로딩·빈·완료 상태
- 다운로드 중 busy 토스트에 `spinner`(회전).
- 확인필요 0 빈 상태 `.rv-empty--ok`: 그린 `check` + "모두 확인했어요".
- 완료 신호 `.export-status`: is-ready 그린 `check` / is-pending 코랄 `alert`.
