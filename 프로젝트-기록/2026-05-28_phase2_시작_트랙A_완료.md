### 11. 인쇄용 PDF 규격 정합 (사용자 v14 실물 PDF 와 비교) ★
- 사용자가 인쇄소에 맡긴 `시제품/도안우미린트리쉐이드_X배너_v14_인쇄용(600mm에 1800mm).pdf` 와 비교
- ★ 발견: **v14 PDF 도 Chromium/Skia-PDF 로 생성** (외주 인쇄 SW 아님). Producer `Skia/PDF m147`, Creator `Chromium`
- ★ v14 실물 사이즈 = **158.8 × 476.3 mm (450×1350 pt)** — 600×1800mm 아님. 인쇄소가 비율 확대 출력
- 어플 초기 PDF 문제 2가지:
  1. 콘텐츠가 158mm 영역에 작게 그려짐 (600px CSS ≠ 600mm)
  2. 2~3 페이지로 분리됨
- ★ 해결: `scripts/export-xbanner-pdf.mjs` 의 검증된 패턴 채택
  - `preferCSSPageSize: true` + banner.css `@page { size: 600px 1800px; margin: 0 }`
  - PDF 출력 시 `html, body { width:600px; height:1800px; overflow:hidden }` 강제 (페이지 분리 방지)
  - viewport 600×1800 px, deviceScaleFactor 1
- 결과: **어플 PDF = v14 와 메타데이터 100% 일치** (158.8×476.3mm / 1페이지 / Skia·Chromium)
- 참고: `2026-05-15_단톡방QR_규칙서_인쇄내보내기.md` 의 "시도 3: preferCSSPageSize 성공" 기록이 결정적
