# 2026-05-29 — UX 전면 개편 Phase 1 (막힌 편집 풀기)

## 배경
Phase 0(미리보기 직접편집 토대) 위에서, 실사용의 가장 치명적 문제 **"틀린 값을 못 고치는 케이스가 절반"** 을 해결.
지금까지 편집 폼은 기본정보·혜택표·결제·프로모션뿐 → 제품(C)·후기(D)·박람회2회(E)·추가혜택(G) 카드는 손도 못 댔고,
카피 톤은 자동 선택만, 영역 순서 조절 없음.

## ✅ 오늘 한 것
### 카드 4종 직접편집 (제품/추가혜택/후기/박람회)
- 미리보기에서 카드(`.product-card`·`.review-card`·`.fair-card`)를 누르면 **그 카드 편집 팝오버**. `decorateEditable` 가 카드에
  `data-edit=card`·`data-kind`·`data-idx` 부여(제품/추가혜택은 같은 `.product-card` 라 state 로 구분).
- `renderCardEditor(kind, idx)` — 종류별 필드 동적 생성:
  - 제품: 사진·제품명·조건·포인트(숫자)
  - 추가혜택: 사진·제품명·조건·뱃지({강조})·공동구매 세대수칩(쉼표, 비우면 뱃지)
  - 후기: 색상(네이버/카페)·아이콘(달력/말풍선)·태그·헤드라인({강조})·설명
  - 박람회: 태그·날짜(줄바꿈)·대상·설명
- 카드별 **+ 카드 추가 / 이 카드 삭제** (종류별 최대 개수 CARD_MAX 에서 추가버튼 자동 숨김).
- {강조} 문법은 "선택한 글자 강조" 버튼으로 숨김(input/textarea 공용 `wrapEmphasis`).

### 제품 사진 라이브러리에서 교체
- 신규 매니페스트 `public/samples/product-images.json` (assets/products 스캔 → **22제품 / 188장**, 제품 라이브러리 JSON 으로 이름 매핑).
- 카드 편집기 "사진 바꾸기" → 제품별 썸네일 그리드 오버레이(`#imgPicker`) → 클릭하면 그 카드 imageSrc 교체. (기획서 약속 실현)

### 카피 톤 3종 선택
- 좌측 "다듬기"에 [금액강조/기간강조/축하형] 버튼. 누르면 큰 헤드라인(maxBenefit before/main/after, 또는 case-d celebration)을
  그 톤 프리셋으로 교체(**금액 amount 는 유지**). 기간강조는 박람회 일수 자동 반영("단 N일간!"). 이후 미리보기에서 미세조정.
- 🐛 **버그 수정(같은 세션)**: 잠실(case-d)은 state 에 maxBenefit·celebration 이 **둘 다** 있고 celebration 만 렌더되는데,
  applyTone 이 maxBenefit 을 먼저 잡아 안 보이는 쪽만 바꿔 "톤이 안 움직임" → **실제로 렌더된 헤드라인**(`.x-banner .celebration` DOM 존재 여부)에 적용하도록 수정.
  재발 방지로 `verify-cards` 에 case-d 톤 변경 체크 추가(16/16).

### 하단 배경 사진 선택 — 없음 / 침실 / 쿠시노 (사용자 요청, 표준 복구)
- 기존 침실 1장 on/off → **없음/침실/쿠시노 3선택**(`assets/full bakground/` 침실 배경_web.jpg·쿠시노 배경.jpg).
- `banner.css .has-bg` 이미지 레이어를 `var(--bg-image, 침실기본)` 으로 변경 → `applyBackground` 이 `data.bgImage`(인코딩 URL)로 교체.
  `state.background`(on/off) + `state.bgImage`(어느 사진). 정적/브라우저 파일이라 서버 재시작 없이 미리보기·내보내기 공통 반영.
- 다듬기 패널 체크박스 → `#bgGroup` 3버튼(`renderBgGroup`/`applyBgChoice`). 검증 `verify-bg.mjs` 13/13(쿠시노 교체 포함)로 갱신.

### 영역 통째로 숨기기 / 다시 보이기 (★ 비파괴 — 사용자 요청)
- "품의서에서 나온 영역이어도 안 보이고 싶을 때"(예: 잠실 후기 이벤트 통째 제거) → `state._hidden`(키 배열)에 표시.
  `orderBlocks` 가 hidden 키를 렌더에서 제외 → **미리보기·내보내기 PDF 둘 다** 빠짐. 데이터는 보존(다시 켜면 원복).
- UI 2곳: ① 좌측 "영역 순서" 목록 각 행 `숨기기` → 아래 "숨긴 영역" 칸으로 이동 + `다시 보이기`. ② 카드 편집 팝오버 푸터 "이 영역 전체 안 보이기".
- 헤더/매장/푸터는 대상 제외(중간 영역만). 검증 `verify-hide.mjs` 10/10 + 격리 렌더 제외 확인 + 회귀 0.

### 영역 순서 드래그앤드롭 (★ 미리보기·내보내기 공통 반영)
- 신규 `render/utils/order.js` `orderBlocks(order, blocks)` — **6개 케이스 전부** 중간 영역을 이 함수로 조립.
  `data._order`(영역 키 배열) 있으면 그 순서로 정렬, 없으면 케이스 기본 순서. → **PNG/PDF 내보내기도 순서 반영**(렌더 레이어).
- 헤더/매장/푸터는 케이스 템플릿에 고정, 중간 영역만(메모리 [[feedback_xbanner_section_reorder]]).
- 좌측 "다듬기"에 **드래그 가능한 영역 리스트**(`#regionOrder`) — 현재 화면 순서를 그대로 표시(`currentRegionOrder` 가 DOM 좌표로 산출),
  드래그로 재배치 → `state._order` 갱신 → 즉시 미리보기 + 자동저장.
