### 3. 단톡방 QR A4 시안 전용 규칙 확정 (사용자 피드백 사이클 4회)

사용자 피드백을 반영하며 단톡방 시안 전용 규칙이 도출됨:

| 제거 | 이유 |
|---|---|
| WELCOME HOME 라벨 | 핵심 정보를 분산시킴 |
| OPEN 큰 글씨 | X배너와 차별화 — A4 단톡방은 후킹보다 명확성 우선 |
| 박람회 기간 | 단톡방은 박람회와 별개로 입주 시즌 내내 운영 |
| 유의사항 안내문 | A4 한 페이지에 핵심만 담기 위해 제외 |

**남긴 영역 (4가지로 단순화)**
1. 헤더 — 위계 반전(★ 채널명이 최대 강조)
   - 1줄: 아파트명(45px·700, 보조 라벨)
   - 2줄: "일룸 전용 입주민 단톡방"(50px·800 코랄 — 배경 없음, 텍스트만)
2. QR 코드 박스 (250×250 + 코랄 5px 보더 + 그림자) + 라벨(20px·700) + 안내(14px·500)
3. 일룸 전용 단톡방 참여 혜택 카드 (헤더 코랄 + 2분할: 혜택 1·2)
4. 참여 방법 카드 (QR코드 스캔 → 단톡방 입장 → 혜택 누리기 / 78px 아이콘)
5. iloom 로고 (하단 가운데)

### 5. QR 코드 — 시제품용 placeholder

- 25×25 SVG grid로 finder pattern + timing pattern + 가짜 data 영역 그림
- 시각적으로만 QR처럼 보임 (스캔은 안 됨)
- → **실제 배포 시 카카오톡 오픈채팅 URL로 진짜 QR 생성 필요** (qrcode.js 라이브러리 또는 API)

### 6. render-banner.mjs 스크립트 개선 — 자동 셀렉터 감지

기존엔 `.x-banner` 셀렉터에 하드코딩되어 있었으나, A4 시제품에도 쓸 수 있도록 자동 감지로 확장.

- 시도 순서: `.x-banner` → `.a4-sheet`
- 본체 실제 크기를 측정해 viewport 재조정 (스크롤바·여백 영향 제거)
- 2x deviceScaleFactor 유지 (인쇄용 고해상도)

→ A4 프로모션 안내, LG 연계 리플렛 등 향후 모든 A4 인쇄물에 동일 스크립트 재사용 가능.

## 🤔 결정한 것

1. **인쇄물 컬러 시스템 일원화** — 한화포레나 골든 스탠다드는 모카·베이지 톤이었으나, 사용자 결정으로 **X배너와 동일한 코랄·크림 톤**으로 통일. 출력물 3종 세트의 시각 일관성 확보.

2. **단톡방 QR 시안 전용 규칙 등록** — 헤더 보조 텍스트(WELCOME HOME, OPEN, 기간)와 유의사항 모두 제외. 핵심 정보(아파트명 + 일룸 전용 입주민 단톡방)에 집중. **메모리 등록 완료**: `feedback_dantokbang_qr_design_rules.md`. 다른 출력물(X배너·A4 프로모션 등)에는 적용 안 함.

3. **혜택 카피 단순화 패턴** — v14 X배너의 공동구매 카드는 "20·40·80·100세대" 칩으로 표현했지만, A4 단톡방에서는 한 줄 카피 "공동구매 세대수별 추가 혜택"으로 압축. 단톡방은 디테일보다 "참여 → 혜택" 신호가 더 중요.

4. **QR 영역은 placeholder 25×25 SVG로 시제품 처리** — 실제 어플에서는 카카오톡 오픈채팅 URL을 입력받아 동적 QR 생성. Phase 2 작업 항목으로 등록.

5. **render-banner.mjs 자동 감지 방식** — 향후 추가될 출력물(A4 프로모션, 단톡방 QR, LG 리플렛 등)이 모두 같은 스크립트로 PNG 생성. 새 셀렉터가 필요할 때만 CANDIDATES 배열에 추가하면 됨.
