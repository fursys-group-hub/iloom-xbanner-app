# 2026-05-29 — UX 전면 개편 Phase 2 (막다른 길 해결)

## 배경
Phase 0(직접편집 토대)·Phase 1(막힌 편집 풀기) 위에서, 실사용의 "막다른 길" 2가지를 해결.
1. **내용 넘침**: 빽빽한 배너(디에이치 등)는 "내용 넘침" 경고만 뜨고 줄일 수단이 없어 글자가 잘릴 위험.
   → 디에이치는 최대 혜택을 최소(0.62)까지 줄여도 **152px 초과**(콘텐츠 1952px).
2. **스캔본/추출 실패**: 글자 없는 PDF(사진 스캔)는 조용히 전부 빈 값, 깨진 PDF·용량 초과는 영문 기술 에러 노출.

## ✅ 오늘 한 것

### (A) 내용 넘침 자동 압축 — 잘리지 않게 안전 영역만 단계적으로 줄임
- **`public/fit-banner.js` 를 `fitBanner(banner)` 단일 진입점으로 재작성.** 렌더 직후 호출.
  1. 최대 혜택 배율(v8 표준 1.0 ~ 최소 0.62) — 그 배너에 맞는 가장 큰 값(기존 fitMaxBenefit 로직).
  2. 최소까지 줄여도 1800px 를 넘치면 → **압축 사다리**를 측정하며 한 칸씩 적용, 1800 안에 들어오는 순간 멈춤.
- **압축 순서**(메모리 [[feedback_xbanner_auto_fill_blank_space]] "콘텐츠 초과 시 압축 순서" 준수):
  유의사항 줄간격/글자 → 그룹 간격 → 결제·카드 간격 → (최후) 매장 박스 패딩·제품 카드 높이.
- **측정 방식**: 배너 클론을 `height:auto` 로 펼쳐 `scrollHeight` 측정(배너는 1800 고정·overflow:hidden 이라 직접 측정 불가).
  클론에 압축값을 누적 적용하며 재측정 → 필요한 최소한만 실제 배너에 인라인 `--var` 로 적용.
- **★ 절대 압축 금지 영역은 CSS 변수로 안 빼서 원천 보호**: 매장명 35px · WELCOME · iloom 로고 · 메인 타이틀 · 아파트명.
  (압축 사다리에 이 변수들이 아예 없음 → 어떤 단계에서도 안 줄어듦.)
- **CSS 변수화**(`public/banner.css`): 압축 대상만 변수로 뺐고 **기본값은 v8 표준과 동일** → 여백 충분한 배너·시제품은 렌더 변화 0.
  - `--gap-group`(60) / `--gap-store-mb`(24) / `--gap-section`(16) — 기존 변수
  - 신규 `--gap-pay`(22) · `--card-gap`(10) · `--product-card-h`(150) · `--store-pad-v`(26) · `--lh-notice`(1.7)
- **미리보기·내보내기 공통**: `app.js`(미리보기)·`preview-app.js`(서버 캡처) 둘 다 `fitBanner` 호출 → **다운로드 PDF/PNG 도 동일하게 안 잘림**.
  (`fitMaxBenefit` 는 `fitBanner` 별칭으로 남겨 하위호환.)
- **결과**: 디에이치 **1952px → 1786px**(✓ 안 넘침), 압축 발동(--gap-group 32px), 매장명 35px 유지.
  대구범어(여백 많음)는 **전혀 안 건드림**(인라인 압축값 없음, 1687px, 최대 혜택 1.0). 서로 무관하게 배너별 독립.

### (B) 스캔본 / 추출 실패 친절 안내 — 기술 에러를 "무엇을 하면 되는지"로
- **`server.js /api/extract`**:
  - PDF 의 의미 있는 글자 < 20자면 **스캔본**으로 보고 `422 {code:'NO_TEXT'}` + "글자가 인식되지 않는 PDF예요… 텍스트 PDF로 다시 올리거나 직접 입력하세요"
  - 추출 중 예외 → `friendlyExtractError()` 가 비밀번호(PASSWORD)/깨짐·PDF아님(INVALID_PDF)/그 외(EXTRACT_FAILED)로 분류해 한국어 안내.
  - **업로드 단계(multer) 오류 핸들러 추가** — 20MB 초과(LIMIT_FILE_SIZE) → `413` + "파일이 너무 커요(최대 20MB)".
- **`public/app.js handlePdfUpload`**:
  - 업로드 전 **PDF 아닌 파일 차단**("PDF 파일만 올릴 수 있어요").
  - 실패 시 `실패: <영문>` 대신 서버의 한국어 안내 + **"직접 입력해서 만들기" 버튼**(`showExtractGuide`) → 시작화면 유지해 막다른 길 방지.
  - 서버 연결 실패(fetch throw) → "서버가 실행 중인지 확인" 안내.
- **`public/editor.css`**: `.status-act` 버튼 스타일(라이트 톤, 코랄 테두리) 추가.

## ⚠️ 운영 메모
- **`server.js` 변경은 서버 재시작 필요.** 이번엔 `--watch` 로 자동 반영됨(엔드포인트 테스트로 확인). `fit-banner.js`·`banner.css`·`app.js`·`preview-app.js` 는 정적(public/)이라 재시작 불필요.
- 압축은 **여백 부족할 때만** 발동(측정 기반) → 기존 시제품/여백 배너 픽셀 변화 0. CSS 변수 기본값이 v8 표준과 동일하기 때문.
- (구버전 UI 의존) `verify-step2/4`·`verify-paymode`·`verify-pay2` 는 여전히 outdated. 현재 권위는 `verify-ux`·`verify-overflow`.
