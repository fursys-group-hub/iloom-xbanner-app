## 🤔 결정한 것

1. **마케팅팀 요청 → 자동 수집으로 전환** — 입주박람회마다 마케팅팀에 요청하는 워크플로우를 자동화. 신제품 출시 시에도 page_id만 알면 즉시 추가 가능.

2. **컬렉트는 키큰옷장(p=62011)으로** — 사용자 직접 요청. "옷장 몸통(컬렉트)"(p=20494)은 부속 위주이고, 키큰옷장이 메인 제품이라 X배너에 적합. 보조 page_id로 부속 페이지 보존.

3. **X배너 어플 자체 npm 환경 유지** — 입문교육 node_modules에 의존하지 않고 X배너 어플 폴더 안에서 자체 실행 가능. 어플의 자체 완결성 유지.

4. **이미지 폴더 한글로 유지** — 시리즈명 그대로 폴더명 사용 (예: `assets/products/컬렉트/`). 매칭 로직에서 영문 변환 없이 동의어사전 활용 가능.

5. **메타데이터(images-meta.json) 자동 생성** — 각 폴더에 alt, parent_text, grand_text 등 텍스트 메타 보존. 자동 매칭 알고리즘이 텍스트 단서 활용 가능.

## 💡 핵심 인사이트

- **카탈로그 풍부도가 페이지별로 다름**: 밴쿠버 30장, 볼케 23장, 스노즈컴팩트 19장 vs 토스티/플렉서블 3장. 단순 제품(매트리스/침대)은 사진 적음, 라이프스타일이 다양한 소파/의자는 많이 받음.
- **신제품(2026/01 출시) 이미지가 알찼다**: 컬렉트 키큰옷장(p=62011)이 일반 컬렉트 부속(p=20494)보다 14장 vs 3장. 신제품 페이지가 카탈로그 풍부함.
- **page_id가 카탈로그에 없어도 직접 검색하면 찾을 수 있음**: 핀(p=50809), 듄(p=46264), 리디(p=56590)는 catalog.json에 없지만 사용자가 직접 사이트에서 찾아줌. 향후 자동 검색 스크립트 추가 가능.
- **마케팅팀 의존성 제거**: 박람회마다 사진 요청하던 워크플로우를 어플이 자동화. 신제품 출시 시 page_id 1개만 추가하면 즉시 라이브러리에 반영.
