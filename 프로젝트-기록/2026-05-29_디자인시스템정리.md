# 2026-05-29 — 편집화면 UI 디자인 시스템 정리 (흰 카드 + 검정/빨강/앰버/그린)

## 배경
사용자가 좋아하는 레퍼런스(깔끔한 흰 카드 모달 + 검정 주요버튼, 빨강은 위험에만)를 보고
**배너 출력물은 그대로 두고** 편집화면(어플 UI)의 모달·버튼·폼·알림을 그 느낌으로 통일 요청.
실시간 리뷰로 추가 지적: 복사 알림 띠 어색 / 패널 답답 / 신뢰도 펼침이 같은 영역 안 느껴짐 /
웜(베이지)·쿨 혼재로 무겁다 / 신뢰도가 카드-인-카드 / 배경사진 라벨 / 발주정보 흰색·회색 의미 없이 다름.

## ✅ 한 것

### 1. 색 토큰 의미별 재설계 (editor.css `:root`)
- 코랄(`--hot`) 1색이 "주요+경고" 겸용이라 의미가 뭉개짐 → **4개 의미로 분리**:
  - `--accent #1C1C1E`(검정)=주요/활성/포커스/진행바, `--danger #E5484D`(빨강)=삭제·오류,
    `--warn #C8841E`/`--warn-deep #9A6411`(앰버)=확인필요·넘침·저신뢰·인쇄경고, `--ok #3E7C4B`(그린)=완료.
- 컴포넌트 규칙을 의미에 맞게 재배정(btn-primary 검정, te-del/card-del 빨강, rv-item/overflow/conf 앰버, export-status 그린 등).
- 배너(banner.css 크림 `--bg-main #FFF5EE`·코랄 `--coral #E55A3D`)는 **불변** — 검증으로 경계 확인.

### 2. 웜→중립 near-white (사용자: "베이지가 섞여 무겁고 답답")
- 따뜻한 크림/베이지 표면을 **중립 near-white**로: `--bg #FAFAFB`/`--surface-2 #F4F4F6`/`--line #E8E8EC`(헤어라인)/`--stage #EBEBEE`. 글자도 중립 `--ink #1F1F23`.

### 3. 버튼·폼·모달·시작화면
- 버튼 위계: 주요=검정 채움 / 보조=흰 카드+선 / **삭제=빨강(`.btn-danger`)** / 주의=앰버 외곽선(`.btn-caution`). 폼 포커스 링 검정.
- 시작화면: "품의서 업로드"=검정, "이어서 편집"=코랄틴트 제거→중립 accent 틴트, 패딩 넉넉.

### 4. 복사/완료 알림 → 간결한 토스트 (사용자: "초록 띠가 어색")
- 화면 폭 가득 찬 `#mainStatus` 초록 띠 **제거** → 우하단 **토스트**(흰 카드+색 점, ok 3.5s·err 6s 자동 사라짐, busy 지속). `showMainStatus`→`showToast` 위임. alert 5곳도 토스트로.

### 5. 네이티브 confirm → 스타일 확인 모달
- `confirmDialog({title,bodyHtml,okText,danger})`(Promise) + `.ui-dialog`(흰 카드, 위험은 빨강 버튼). 다운로드 안전확인/새배너/기본값복원 confirm 교체.

### 6. 신뢰도 readout = 표 형태 (사용자: "카드 안 카드라 답답")
- 펼치면 summary(헤더)+목록이 **하나의 흰 카드 + 행 구분선(표)**. 행마다 흰 카드 중첩 제거 → 깔끔.

### 7. 배경 사진 크게 + 라벨 알약
- `.swatch-grid--bg` 2열 가로형(4:3) 큰 카드로 사진 한눈에. "침실/쿠시노" 라벨은 하단 흰 바 → **좌하단 알약 뱃지**.

### 8. 색 의미 일관화 (사용자: "왜 흰색·회색이 의미 없이 다르냐")
- **흰색=자동·읽기전용 / 회색=직접 입력칸 / 앰버·빨강=확인필요·문제** 규칙 확립. `발주 정보` 자동값(`.order-auto`)을 회색→**흰색**으로 통일(읽기전용이므로).

### 9. 패널 여백 (답답함)
- 사이드바 패딩 28/26, rv-block 30/26, ed-field 18 등 여백 리듬 확대.
