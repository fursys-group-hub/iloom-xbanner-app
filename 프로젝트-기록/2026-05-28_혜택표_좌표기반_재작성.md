# 2026-05-28 — 혜택표 추출 좌표 기반 재작성

## ✅ 오늘 한 것

혜택표 추출(`app/extract/parsers/benefit-table.js`)을 **텍스트 토큰 개수 방식 → x좌표 컬럼 클러스터링 방식**으로 전면 교체했다.

### 왜 바꿨나 (문제)
기존 방식은 한 줄(line)의 텍스트를 토큰으로 쪼개 "가장 흔한 토큰 개수와 같은 행만" 채택했다. PDF 표에서 빈 셀·세로병합·토큰분리가 생기면 행마다 토큰 수가 달라져 일부 행이 통째로 누락됐다.
- **잠실**: 5행 중 3행만 추출(200/500 누락). 추가혜택 컬럼(세로병합)도 인식 못 함.
- **디에이치 700행**: "700만원"이 "700"+"만원" 두 토큰으로 분리돼 버킷 감지 실패 → 4행만.
- **한화/대구/창원**: 표 안 각주 산문("추가혜택(박람회전용)의 경우 본사:대리점=5:5" 등)이 별도 컬럼으로 오인 → 컬럼 수 +1, 총합 잘못 합산(창원 1000행 200만원으로 부풀려짐).

### 새 로직 (좌표 기반)
1. **버킷 토큰 수집** — "N00만원" 토큰들을 x좌표와 함께 모음
2. **금액 컬럼 x 확정** — 가장 흔한 x 밴드 = 구매금액 위치 (본문 잡문 "참가비 200만원"은 x가 달라 배제)
3. **데이터 행 선정** — 버킷이 금액 컬럼 x ±25 인 줄만
4. **컬럼 중심 클러스터링** — 데이터 행의 "값 토큰"만 x로 1D 클러스터링(gap 40) → 4/5/6컬럼 자동
5. **셀 배정** — 각 값 토큰을 가장 가까운 컬럼 중심에 배정. 빈 셀은 자동 "—"
6. **세로 병합 전파** — 한 행에만 값 있고 나머지 전부 빈칸인 컬럼은 병합셀로 보고 전 행에 전파 (잠실 추가혜택 2만원)
7. **총혜택 자동 합산** — total 칸 없으면 기존+추가 등 합산 (수원성중흥/부산센텀)

### 핵심 헬퍼 3종 (신규)
- `mergeAdjacentItems(items, maxGap=6)` — 붙어 쪼개진 토큰("700"+"만원") 복원. 셀 간격(60px+)보다 훨씬 작은 6px만 병합해 옆 셀 오염 없음.
- `isValueToken(s)` — 금액/숫자/분리된 "원"/dash 만 인정. 한글 각주 산문을 컬럼 계산·배정 전에 제거.
- `clusterX(xs, gap)` / `propagateMergedColumns()` / `normalizeCell()`
