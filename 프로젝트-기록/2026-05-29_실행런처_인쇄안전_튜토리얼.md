# 2026-05-29 — 실사용 마무리: 원클릭 실행 + 인쇄 전 안전 강화 + 첫 사용 튜토리얼

## 배경
미리보기 직접편집 개편(Phase 0~3) 후 실사용자(비개발자·바쁜 영업담당) 관점 상세 감사 → 기능 외 3가지가 남음:
어플을 못 켬(터미널 전용) / 넘침·저해상도여도 다운로드를 안 막음 / 처음 쓰는 사람이 "클릭해 편집"을 모름.
사용자가 이 3개 선택 + "처음에 튜토리얼처럼 눌러보며 배우게" + "신뢰도 원문은 접힌 채로" 요청.

## ✅ 한 것

### 1. 원클릭 실행 런처
- 신규 `실행.bat`(앱 폴더 루트) — Node 설치 확인(없으면 nodejs.org 안내) → 2초 뒤 기본 브라우저로 localhost:3000 자동 열기(숨김 PowerShell) → `node server.js`.
- 비개발자가 **더블클릭만으로** 어플 실행. 검은 창을 닫으면 종료. 이미 실행 중이면 EADDRINUSE 나도 브라우저는 열려 OK.
- 한계: Node 최초 1회 설치·사내 호스팅은 IT 협의(범위 밖) — bat 안내문으로 갈음.

### 2. 인쇄 전 안전 강화
- `download()` 의 확인창을 빈 필수항목 only → **종합 점검**으로: `validationWarnings()`(빈칸 + 넘침 + 저해상도<100DPI)를 한 번에 모아 강한 confirm("인쇄 전에 확인해 주세요: … 이대로 인쇄하면 잘리거나 흐리게 …"). recheckImages() 선행 호출로 최신화.
- 경고 가시성: 문제 있으면 PNG/PDF 버튼에 주의 상태 `.btn-caution`(코랄 테두리). done-warn 경고는 내보내기 버튼 위에 그대로.

### 3. 첫 사용 튜토리얼(스포트라이트) + 완료 신호
- 신규 `public/tour.js` — 딤 오버레이 + 대상 스포트라이트(box-shadow 홀) + 말풍선. 4단계: 미리보기(클릭해 편집)→확인필요→다듬기→내보내기. [다음]/[건너뛰기], ESC 종료, 끝나면 `localStorage['iloom-xbanner-tour-done']`.
- 첫 진입(업로드/새로작성/이어서편집 직후) 1회 자동(`maybeStartTour`). **자동화(navigator.webdriver)에선 자동 시작 안 함** → 기존 Playwright 검증 무영향. 툴바 **"도움말"** 버튼으로 언제든 재실행(`startTour`).
- 대상엔 `data-tour`(preview/review/polish/export) 속성만 부여(기존 요소 무수정).
- **완료 신호**: 내보내기 상단 `#exportStatus` — 확인필요 0 + 경고 0 → 초록 "다 확인했어요 — 인쇄해도 좋아요", 남으면 코랄 "확인할 게 N개 남았어요". `renderDoneSummary` 끝에서 `renderExportStatus` 호출.

### 4. 제약 — 신뢰도 원문 접힌 채
- `#extractReadoutWrap`(`<details>`)는 `open` 자동 설정 안 함 → 업로드 후에도 접힌 상태 유지(검증으로 확인). 튜토리얼도 강제로 열지 않음.
