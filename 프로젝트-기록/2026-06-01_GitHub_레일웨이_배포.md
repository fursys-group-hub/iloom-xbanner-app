- 저장소: **https://github.com/sujiui/iloom-xbanner-app** (🔒 Private)
- `git init` → 커밋 → `gh repo create --private --push`
- 새 파일: `README.md`(앱 소개·실행법·폴더구조), `.gitignore`

  - `*.mp4`, `*.mov` — 191MB 화면 녹화 영상이 GitHub 100MB/파일 제한 초과
  - `_tmp-payload.json`, `app/scripts/output/`(스크린샷 출력물), OS/에디터 파일

### 2. 레일웨이 배포 설정 파일 추가
- **Dockerfile** (저장소 루트): `node:20-bookworm-slim` 기반
  - Ghostscript 설치 (인쇄용 CMYK 변환 — 없으면 RGB 폴백)
  - `app/`에서 `npm ci --omit=dev`
  - `npx playwright install --with-deps chromium` (Chromium + OS 라이브러리)
  - 저장소 **전체** COPY (server.js가 상위 `../assets/` 참조하므로 app/만으론 부족)
  - `CMD ["node", "app/server.js"]`
- **.dockerignore**: node_modules·mp4·임시파일·.git 제외 (빌드 컨텍스트 축소)
- **railway.json**: DOCKERFILE 빌더 명시 + 헬스체크 `/api/health` + ON_FAILURE 재시작

### 3. 레일웨이 배포 실행
- `railway login`(사용자 브라우저 인증) → `railway init --name iloom-xbanner-app`
- `railway up --detach`로 업로드·빌드 트리거 → `railway domain`으로 공개 주소 생성
- **공개 주소: https://iloom-xbanner-app-production.up.railway.app**
- 워크스페이스: sujiui's Projects / 계정 suzzz13134@gmail.com

## 🤔 결정한 것
- 저장소는 **비공개(Private)** — 사내 제품 이미지·배경·자료 보호
- 호스팅은 **레일웨이** + **Dockerfile 방식** (Nixpacks 아님)
  - 이유: Playwright Chromium은 정적 호스팅(Vercel/Netlify) 불가, 시스템 라이브러리가 필요해 Docker가 가장 안정적
- 배포 방식은 **`railway up`(로컬→레일웨이 직접 업로드)** 사용
  - `.gitignore` 적용되어 node_modules·mp4 제외되고 assets(86M)만 업로드됨

- 레일웨이는 사용량 기반 과금 — 트래픽 없으면 비용 거의 없음. 대시보드: railway.com
