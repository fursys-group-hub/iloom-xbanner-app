## 🐢→⚡ 배포 후 속도 최적화 (같은 날)
- **증상**: 시작 화면이 29초+ 느리게 뜸
- **원인 2가지**:
  1. 레일웨이 서버 지역 **US West**(한국에서 먼 거리)
  2. **배경 사진이 초대형** — 침실 23MB(5069×7603px)·레마 14MB·필즈 17MB. 게다가 app.js:953 썸네일이 원본을 그대로 로드 → 시작 화면에서 30~50MB
  - 실측: 침실 배경 다운로드 **29.6초**
- **해결**: sharp로 배경 3장 리사이즈(긴 변 3000px·q82 mozjpeg) → **23MB→0.28MB(99%↓)**. 원본은 `assets/_배경_원본_백업/`에 보관(gitignore+dockerignore로 배포 제외)
  - 인쇄 영향 없음: 배너는 600px폭 렌더→인쇄소 3.78x 확대라 ~2300px면 충분. 2000~3000px 유지
- **결과**: 침실 **29.6초→1.6초**, 레마 1.3초·필즈 1.5초. PNG 출력 정상 재확인
- **남은 여지(선택)**: 서버 지역을 Singapore(asia-southeast)로 옮기면 잔여 ~1.5초 지연(US↔한국)이 ~0.3초로. 레일웨이 대시보드 Service→Settings→Region에서 변경(재배포 발생)
- 재배포: `railway up`

## 🔧 매장명 추출 보강 (같은 날)
- **문제**: 품의서 13건 전수 테스트 중 **대전 하늘채 스카이앤**이 매장명(서대전점)을 못 뽑아 매장 박스가 통째로 비고 큰 공백 발생
- **원인**: 이 PDF는 매장 표기가 기존 3규칙과 다름 — 제목 괄호 `(서대전점)` / "진행 **대리점**"(사전엔 "진행 매장"만) / `'서대전점'` 따옴표(기존 규칙은 `'일룸 OO'` 필요)
- **해결**: `parseStoresFromTitle()` 추가 — 제목 괄호 안 `(OO점)` 추출. 거의 모든 품의서 제목에 있어 가장 보편적인 **최종 폴백**(라벨/따옴표/번호 다 실패할 때만). `extract/parsers/basic-info.js`
- **검증**: 품의서 13건 매장 추출 **13/13**(대전하늘채=`[제목괄호매장]`, 나머지 12건 원래 방식 그대로 회귀 없음). 정답데이터 88.2% 유지. 대전하늘채 렌더 → "일룸 서대전점" 35px 정상
- **참고**: 전수 테스트 스크립트 `app/scripts/_test_all_pdfs.mjs`(13건 일괄 추출·렌더·스샷), `_test_delcard_redistribute.mjs`(카드 삭제 재분배). 둘 다 미커밋(로컬 도구)
- 추출 정확도는 여전히 평균 88% — 매장/객단가 등 일부 필드는 미리보기 직접수정 전제([[feedback_xbanner_draft_then_customize]])

## 🔐 로그인 + 저장/갤러리/공유 (2단계, Supabase)
- **DB**: Supabase iloom-lsa 공유 인스턴스, public 스키마 `xbanner_users`/`xbanner_banners`(접두 분리). schema.sql 은 `app/lib/schema.sql`. RLS on(서버는 service_role 로 우회)
- **인증**: 이름→비밀번호 2단계(처음이면 가입). bcryptjs 해시 저장, 세션은 HMAC 서명 토큰(무상태, SESSION_SECRET). 가입 정책=사내 자유 가입
- **서버**: `app/lib/store.js`(Supabase REST + 토큰) + server.js 라우트 `/api/auth/login`·`/api/auth/me`·`/api/banners`(목록/열기/저장/삭제)·`/api/share/:id`·`/s/:id`
- **프론트**: 툴바 저장/내배너함/로그인·로그아웃, 로그인·제목 다이얼로그, 갤러리(저장 data로 미니 미리보기 렌더 썸네일), 공유 링크 복사. `public/share.html`=읽기전용 뷰어
- **비밀키**: `app/.env`(SESSION_SECRET·SUPABASE_URL·SERVICE_ROLE_KEY) — .gitignore+.dockerignore 로 깃·이미지 제외, Railway 환경변수로 주입(railway variables). 로컬 실행은 `node --env-file=.env server.js`
- **검증**: 백엔드 직접(가입/재로그인/틀린비번/토큰위조/저장·목록·열기·공유·삭제) + 브라우저 E2E(로그인다이얼로그·저장·갤러리썸네일·공유페이지·계정라벨·토큰유지) 전부 통과, 콘솔에러 0. 테스트 계정 정리 완료
