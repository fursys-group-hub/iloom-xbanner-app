# 2026-05-28 — 어플 UX 위저드(단계형) 재설계

## 배경
사용자가 "한 화면에 하나만(One Thing Per Page)" UX 원칙(브런치 https://brunch.co.kr/@chulhochoiucj0/31)을 근거로,
사이드바에 모든 편집을 한꺼번에 담던 기존 구조가 비개발자 영업담당에게 복잡하다고 지적 →
**시작화면(선택지 2개) → 단계별 편집(다음/다음)** 흐름으로 재설계 요청.

## 확정 방식 (사용자 선택)
**스텝 + 상시 미리보기** (절충형) — 왼쪽은 단계별로 바뀌고, 오른쪽 X배너 미리보기는 항상 표시.
(원칙 100%인 "완전 한 화면씩"은 편집 중 실시간 미리보기를 잃어 제외)

## 구현
### 시작화면 (오버레이)
- `#startScreen` 전체 오버레이 + 중앙 카드. 버튼 2개: **「품의서 업로드」**(파란 주버튼) / **「전부 새로 작성」**.
- 업로드: 클릭 파일선택 + 카드에 드래그앤드롭. 분석 성공 시 `hideStart()` → 1단계로.
- 새로 작성: 예시(case-a) 로드 → 1단계로.

### 단계 (위저드)
- 단계: ① 기본 정보 ② 혜택표 ③ 결제 ④ 특별 프로모션 ⑤ 완료
- `STEPS` 배열 + `visibleSteps()` — **프로모션 단계는 `specialPromoSection.cards` 있을 때만(case-b)** 자동 노출/스킵.
- 단계 표시기(`.step-chip`): 현재=파랑, 지난 단계=초록, 클릭 시 해당 단계로 점프.
- 푸터: [← 이전] / [다음 →] (1단계엔 이전 숨김, 완료엔 다음 숨김).
- 완료 단계: PNG/PDF 다운로드 + "처음으로(새 배너)". 미리보기 상단 툴바에도 PNG/PDF 상시.
- 툴바 "← 처음" 버튼으로 언제든 시작화면 복귀.

### 보존
- 기존 편집기 로직(혜택표 셀/행, 결제 체크박스, 프로모션 카드, PDF 추출, 다운로드, 신뢰도 테두리)은 그대로. 위에 네비게이션만 얹음.
- UI 무이모지 규칙 준수(상태문구 ⏳/✅/❌ 도 제거, 색상 클래스로 위계).

### 파일
- `public/index.html` — 시작화면 + step-panel + step-nav + step-footer 구조
- `public/editor.css` — 시작화면/스텝칩/단계패널/푸터/링크 스타일
- `public/app.js` — STEPS·goToStep·renderStepNav·stepNext/Prev·showStart/hideStart + 시작/단계 이벤트 배선

## 검증 (Playwright 실브라우저, 콘솔 에러 0)
- 시작화면 표시 + 뒤에 미리보기 렌더됨 / 「새로작성」 → 시작화면 닫힘 → 1단계
- case-a: 단계 4개(프로모 없음). 다음 → 혜택표(미리보기 유지). 칩 점프(결제, 체크박스 3). 완료 단계(다음 숨김, PNG2 표시)
- case-b(한화 업로드): 시작화면 닫힘 + **단계 5개(프로모션 포함)** + 프로모션 카드편집기 4개 + 미리보기 상시

## ⚠️ 운영 메모 (중요) — 서버 재시작 / 코드 반영
**증상**: aptName 등 추출 코드를 고쳤는데도 화면(업로드)에선 옛 값("전라남도 순천시")이 계속 나옴.
**원인**: 실행 중인 서버(`node server.js`)는 시작 시 추출 모듈을 메모리에 올려두고 계속 사용 → **코드 수정이 반영 안 됨**. (별도 `node 스크립트.mjs` 검증은 매번 새로 import 해서 깨끗하게 나오므로 "테스트는 통과, 실제 앱은 옛 값" 불일치 발생.)
**해결/예방**:
- 추출/렌더 코드 수정 후엔 **서버 재시작 필수**. 앞으로는 `node --watch server.js` 로 띄워 자동 반영.
- **실행 위치/명령**: `cd "3. 일룸 인쇄용X배너 어플/app" && node --watch server.js` (서버 본체는 `app/server.js`, 포트 3000). ※ 루트 CLAUDE.md "## 3" 섹션의 옛 안내(`node server.js`, templates.js 구조)는 구버전 — 현재는 `app/` 모듈 구조.
- 검증은 **isolated 스크립트가 아니라 실제 `/api/extract` 엔드포인트로** 확인할 것. (서버 캐시/실앱 동작을 그대로 반영)

## ✅ 실제 업로드 경로(/api/extract) 정확도 검증 — 재시작 후
서버 최신화 후 실제 업로드로 이전 문제건 전수 확인 (전부 정상):
- 순천 → "순천 트리마제" / 교대역 → "교대역 모아엘가 그랑데" / 양정 → "양정 자이 더샵 SK뷰" / 부산센텀 → "두산위브더제니스 오션시티"(매장 3개) / 익산 → 이름 + 총혜택 100만원(오기 자동수정)
- ※ aptName 정제는 일반 규칙(주소·재개발 제거)이라 신규 품의서도 같은 패턴이면 자동. 추출은 휴리스틱이라 100%는 아님 → 빗나가면 단계별 수동 보정으로 현장 수정(설계 의도).
