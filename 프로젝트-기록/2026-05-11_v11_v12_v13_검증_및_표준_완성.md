### 5. 케이스 자동 분류 if-else 명세 ★ NEW
```javascript
function classifyCase(data) {
  if (!data.박람회.참여여부) return 'F';
  if (data.기본정보_2아파트?.length === 2) return 'D';
  if (data.박람회.기간_차수?.length >= 2) return 'E';
  if (data.특별프로모션?.length >= 3) return 'B';
  const products = data.특별프로모션?.filter(p =>
    ['옷장단독', '모션', '소파', '학생방'].includes(p));
  if (products?.length >= 1) return 'C';
  return 'A';
}
```

## 🤔 핵심 결정 사항

### 1. 매장명 35px 절대 보호 원칙
v8 → v13까지 매장명 사이즈가 들쭉날쭉했던 게 가장 큰 문제. "다른 영역 압축하더라도 매장명은 절대 35px 유지" 표준화.

### 2. 결제 안내 3단계 모드 도입
컴팩트(70px) ↔ v8 표준(280px) 둘 사이 빈 공간 200px 활용을 위해 카드 2(150px) 신규 추가. 사용자 토글로 선택 가능.

### 3. 사용자 토글 + 자동 알고리즘 균형
"빈 공간 무조건 채우는 게 답이 아니다" — v10 검증에서 사용자가 컴팩트 모드의 여백의 미 선호. auto / compact / card_1 / card_2 4가지 토글로 사용자 선택권 확보.

### 4. 영역 순서 조절 (제품 카드 → 혜택표 위)
v13 사용자 피드백: 제품 → 가격 순서가 더 자연스럽다. 케이스 C 디폴트 순서 업데이트.

### 5. 케이스 자동 분류 알고리즘
PDF 추출 데이터에서 어떤 패턴이면 어느 케이스로 분류할지 if-else 명세 확정. 9건 정답 데이터 모두 자동 분류 가능.
