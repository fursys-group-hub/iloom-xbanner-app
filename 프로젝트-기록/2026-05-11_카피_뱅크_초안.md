## ✅ 오늘 한 것

- bash unzip + Node.js 정규식으로 일괄 처리
- **6개 PPTX, 총 21개 슬라이드** 분석 완료

| 오산 세교 파라곤 (4장) | X배너 + A4 POP + 후기이벤트 | 금액강조 + 축하형 + 후기 |

### 3) 3가지 톤 분류 + 자동 추천 규칙 설계

| 톤 | 언제 | 적용 사례 | 예시 카피 |
|---|---|---|---|
| **금액강조** | 박람회 2일 + 혜택 100만원+ (가장 보편) | 광주데시앙, 창원, 오산세교, 디에이치 | "입주박람회 현장에서만 드리는\n최대 {최대혜택} 특별 혜택\n지금 바로 만나 보세요!" |
| **기간강조** | 박람회 3일+ 또는 박람회 미참여 (LG연계 등) | 인덕원(5일), 디에이치(3일), 수원성중흥(1개월) | "일룸에서\n입주민을 위한\n{일수}일간의 특별한 혜택을\n만나보세요." |
| **축하형** | X배너 2번째 시안, 공동공략 매장 소개 | 광주데시앙 슬2, 오산세교 슬2, 평택 | "{아파트명} 입주를\n진심으로 축하드립니다" |

### 4) 변수 시스템 설계
어플 자동 추출 결과를 카피에 채워넣는 표준 변수:
- `{아파트명}`, `{아파트약명}`, `{매장명1}`, `{매장명2}`, `{매장명_연결}`
- `{최대혜택}`, `{기간_시작}`, `{기간_종료}`, `{일수}`, `{매장연락처}`

### 5) 산출물 — `카피_뱅크.json` 작성
- **톤별_카피**: 3톤 × (원본 + 변형 + 서브헤드라인)
- **용도별_카피**: 환영문구 / 메인타이틀 / 매장안내 / 혜택소개 / 혜택표헤더 / 결제안내 / 유의사항
- **특별_프로모션_카피**: 동의어사전의 7가지 키워드(세트구매/옷장단독/모션/소파/학생방/한정프로모션/단톡방공동구매) → 카피 풀 매칭
- **후기_이벤트_카피**: 5가지 (네이버예약/입주카페/단톡방/LSA우수후기/특별홍보)
- **구축아파트_PICK_시리즈**: 부산사업소 9개 매장 A4 POP 양식
- **_자동_추천_규칙**: 어플이 톤 결정하는 if-else 규칙
- **_사용_예시**: 광주 데시앙 케이스로 변수 채우기 → 톤 결정 → X배너 조립 흐름

## 🤔 결정한 것

1. **3개 톤은 케이스별 자동 결정** — 박람회_참여/일수/혜택금액으로 if-else. 사용자가 추출 결과 화면에서 톤만 바꾸면 카피 전체가 같이 바뀌도록.

2. **유의사항은 표준 10종 풀에서 자동 선택** — 5컬럼/4컬럼 혜택표 + 단톡방 운영 여부 등에 따라 어플이 적합한 유의사항만 자동 선택해 카피 사용.

3. **PICK 시리즈는 별도 카테고리** — 박람회용 X배너와 구축 아파트 공략용 A4 POP 양식이 본질적으로 달라서 분리. 부산사업소 9개 매장 예시 그대로 보존.

4. **특별 프로모션 매핑은 동의어사전을 거쳐 자동** — 품의서 → 동의어사전(키워드 7종 감지) → 카피뱅크(특별프로모션_카피 섹션) → X배너에 추천 카피 자동 배치.

5. **혜택표 헤더는 컬럼 수별 4가지 버전** — 4컬럼(구버전 2025) / 5컬럼(신버전 2026) / 6컬럼(잠실 특수) / 단순 4컬럼(부산사업소 PICK 단순형). 어플이 자동 감지해 헤더 자동 적용.

## 💡 핵심 인사이트

- **광주 데시앙이 가장 정교한 표준 시안** — 슬라이드1(박람회) + 슬라이드2(축하)로 X배너 2종을 한 케이스에 묶음. 이 구조를 어플 표준으로.
- **부산사업소 PICK 시리즈는 별개** — 박람회 시각물과 다른 워크플로우 (구축아파트 매장별 일괄 자료). 향후 PICK 전용 템플릿이 필요할 수 있음.
- **인덕원 시안에서 발견** — "5일간의 특별한 혜택을 만나보세요" 라는 기간강조 카피가 박람회 1일~5일 모두 적용 가능한 만능 패턴.
- **유의사항은 9건 거의 동일** — 매장명만 바뀌고 본문은 표준. 어플에서 자동 생성 가능.
- **결제 안내(Npay/L.POINT/SSGPAY)는 9건 전부 동일** — 상수로 고정해도 무방.
