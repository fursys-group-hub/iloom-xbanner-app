# 2026-05-28 — 매장(stores) 추출 갭 해결

## ✅ 한 것
케이스 C(디에이치)·D(잠실)에서 매장 박스가 비어 보이던 문제 해결. 가장 임팩트 큰 공통 갭이었음.

### 원인
- `공략매장` 라벨(공략 매장/진행 매장/…)이 디에이치·잠실 PDF에서 **매칭 실패** → stores 빈 배열
- 게다가 매장명이 "프리미엄샵 논현" / "송파"·"강동아이파크"처럼 **"점" 없는 형식**이라 `parseStores`의 'X점' 패턴으로도 못 잡음

### 해결
1. **따옴표 폴백** (`basic-info.js parseStoresFromQuotes`): 공략매장 라벨이 비면 "…'일룸 프리미엄샵 논현' …에서만 단독 홍보 진행" 줄에서 따옴표 안 '일룸 XXX' 매장명을 추출. 이 줄은 품의서 공통 패턴.
2. **2개 이상이면 "프리미엄샵" 접두어 제거** (`to-app-state.js`): 단일 매장은 풀네임 유지("일룸 프리미엄샵 논현"=v10), 2개 이상은 지점명만("일룸 송파"·"일룸 강동아이파크"=v11) → 줄바꿈 방지 + 중복 축약.

### 결과 (6케이스 검증, 회귀 없음)
| 케이스 | stores |
|---|---|
| 디에이치 | 일룸 프리미엄샵 논현 (v10 일치) |
| 잠실 | 일룸 송파 / 일룸 강동아이파크 (v11 일치) |
| 한화포레나 | 일룸 서대전점 / 일룸 대전둔산점 (기존 라벨 경로, 그대로) |
| 대구범어 | 일룸 수성점 |
| 교대역 | 일룸 광천점 / 일룸 봉선점 |
| 창원 | 일룸 창원점 |

→ 디에이치/잠실 자동 플로우에서 매장 박스 정상 렌더 확인 (잠실은 줄바꿈 없이 한 줄씩).

# 잠실 헤더 마무리 — aptName "아이파크" + 이중 기간 (이어서)

## ✅ 한 것 (basic-info.js 다중 아파트 보정 블록)
잠실(2아파트)에서 헤더가 시제품 v11 과 완전 일치하도록 자동 추출 보강.

1. **풀 아파트명**: "아래와 같이 **잠실래미안 아이파크 & 잠실르엘** 입주아파트를 공략" 인트로 줄에서 `&`/`·`/`,` 로 분리 → "잠실래미안 아이파크 · 잠실르엘" (단일 아파트는 이 줄에 이름 1개라 미발동 → 기존 라벨 유지)
2. **이중 기간(periodDetail)**: 아파트별 "※ 입주 박람회 현장 계약 혜택 : 11/22(토)~11/23(일)" / "11/28(금)~11/30(일)" 2줄을 순서대로 추출, 약명 단축명(잠실래미안/잠실르엘)과 zip → `[{label:'잠실래미안',range:'11.22~23'},{label:'잠실르엘',range:'11.28~30'}]`
3. **전체 기간**: 첫 박람회 시작 ~ 마지막 박람회 끝 = 2025-11-22 ~ 2025-11-30 (연도는 기존 추출 period/작성일에서 상속)
4. to-app-state: `fields.periodDetail` → `state.periodDetail` 전달 (Header 가 렌더). 기존 약명 paren aptName 오버라이드는 제거(basic-info가 더 정확).

## 🔬 검증
- 잠실 자동 플로우: aptName "잠실래미안 아이파크 · 잠실르엘" / period 11.22~11.30 / periodDetail 이중 기간 / 매장 2개 → **헤더 v11 완전 일치**
- 단일 아파트(디에이치/한화/대구범어): periodDetail 없음, aptName/period 기존 그대로 (isMulti=false → 미발동)
