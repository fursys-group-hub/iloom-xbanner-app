# 2026-05-15 — 단톡방 QR 규칙서 완성 + 인쇄용 내보내기 스크립트

## 작업한 것:

### 1. 단톡방 QR A4 디자인 규칙서 완성 (16개 섹션)
- 기존 v1 시제품(도안 우미린)을 분석하여 **품의서만 전달하면 동일한 시안이 나오는** 완전한 규칙서 작성
- 메모리: `feedback_dantokbang_qr_design_rules.md`

### 2. 창원 롯데캐슬 포레스트 테스트 시안
- 품의서(일룸-품의26-01-00262)에서 혜택 추출 → 시안에 반영
- 규칙서 검증: 아파트명·QR·혜택만 바꾸면 동일 퀄리티로 생성됨을 확인

### 3. 규칙서에 반영된 주요 결정사항
| 항목 | 확정값 |
|------|--------|
| 시트 패딩 | `70px 60px 50px` (상·좌우·하) |
| QR 박스 padding | 0 (SVG 자체 margin 4로 여백 확보) |
| QR SVG 생성 | `--margin 4` 고정 |
| 혜택 내용 | **품의서에서 추출** (다른 아파트 복사 금지) |
| 혜택 1개일 때 | 라벨/시점 제거, 본문 한 줄 통합, 헤더와 중복 문구 제거 |
| 레이아웃 | QR 위아래 flex:1 spacer → 자동 중앙 / 혜택+참여방법+로고 하단 묶음 고정 |
| 헤더 margin | 없음 (spacer가 담당) |

### 4. 인쇄용 내보내기 스크립트 2종 작성
| 출력물 | 스크립트 | 형식 |
|--------|---------|------|
| 단톡방 QR A4 | `scripts/export-a4-png.mjs` | 고해상도 PNG (2382x3369px, 300dpi) |
| X배너 | `scripts/export-xbanner-pdf.mjs` | 벡터 PDF (1페이지) |

### 5. 인쇄 내보내기 조사 결과
- Chromium(Playwright)은 CSS `@page { marks: crop; bleed: 3mm }` 미지원
- 도련/재단선을 HTML로 직접 그리는 방식은 Playwright PDF 렌더링 한계로 참고자료 수준 불가
- **결론**: 단톡방 QR → 고해상도 PNG 전달, X배너 → 벡터 PDF 전달 (업체가 도련 자체 처리)

### 6. X배너 벡터 PDF 해결 과정
- 시도 1: `zoom: 3.78` → 페이지 4분할 (실패)
- 시도 2: `transform: scale(3.78)` + 뷰포트 2268x6804 → 오른쪽 잘림 + 3페이지 (실패)
- 시도 3: `preferCSSPageSize: true` + `@page { size: 600px 1800px }` → 1페이지 벡터 PDF 성공
- 벡터이므로 인쇄업체가 600x1800mm로 확대해도 선명

## 결정한 것:
- 단톡방 QR 시안은 품의서 기반으로 자동 생성 가능
- 인쇄업체 전달: 단톡방 = PNG + "A4 단면", X배너 = PDF + "600x1800mm"
- PDF 내보내기 스크립트(export-a4-print.mjs)는 삭제 → PNG만 사용
