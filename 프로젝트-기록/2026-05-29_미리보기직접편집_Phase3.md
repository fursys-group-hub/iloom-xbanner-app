# 2026-05-29 — UX 전면 개편 Phase 3 (신뢰·검수 강화)

## 배경
Phase 0~2 로 "직접편집 토대 / 막힌 편집 / 막다른 길"을 풀었고, 이제 **"자동으로 채운 걸 믿고 검수"** 단계.
비개발자 영업담당이 "이 값이 품의서 어디서 왔는지, 맞는지" 확신하고 인쇄로 넘기게 하는 게 목표.

## ✅ 오늘 한 것

### (A) 추출 원문 보기 — 편집 팝오버 상단 "품의서에서 가져온 내용"
- 영역을 클릭해 편집 팝오버를 열면, 맨 위에 그 영역이 **품의서 어느 줄에서 왔는지(raw)** + 친절 신뢰도 칩을 보여줌.
  예) 제목·기간 편집 → "정확 · 아파트명 품의서: 디에이치 방배" / "확인 권장 · 박람회 기간 품의서: ※ 입주 박람회 현장 계약 혜택 …".
- `app/public/app.js` `renderEditSource(panel, isCard)` + `PANEL_SOURCE`(패널→추출필드) + `index.html` `#edPopSource` + `editor.css .ed-source/.src-*`.
- 카드(제품/후기/박람회) 편집 시엔 프로모션 추출 결과를 원문으로.

### (B) 신뢰도 표시 확대 — 좌측 "품의서에서 읽은 값" readout (접이식)
- "확인이 필요해요" 아래에 **품의서에서 읽은 값 — 신뢰도 보기**(`<details>`). 각 줄 = 친절 신뢰도 점 + 라벨 + 원문/값.
  - **배너에 보이는 항목**(아파트명·기간·매장·혜택표·유의사항) = "고치기" 태그 → 클릭하면 미리보기로 점프 + 편집.
  - **참고 항목**(객단가·약명·세대수·입주월·프로모션·결제) = "참고" 태그(읽기 전용) — 배너엔 안 나오지만 추출 신뢰도를 확인.
- 객단가/약명/세대수/입주월은 배너 미표시 값이라 **막다른 길 방지 위해 읽기 전용 "참고"**로만 노출(Phase 2 교훈: 못 고치는 검수 항목 금지).
- 서버: `extract/to-app-state.js` `_extraction.fields` 에 **promotions·payment 신뢰도+원문 추가**(기존엔 누락). 객단가(pricePerCustomer)/약명(shortName)/세대수(households)/입주월(occupationMonth)은 이미 meta 에 있었음.
- `app/public/app.js` `renderExtractReadout()`/`READOUT` + `index.html #extractReadoutWrap` + `editor.css .readout-*/.ro-line-*`.

### (C) 프로모션 카드 확인 — 검수 리스트 + 미리보기 형광
- 자동 추출 카드(프로모션/제품/후기/박람회)의 조건·포인트가 불확실(신뢰도 확인 권장/필요)하고 실제 카드가 있을 때만
  "확인이 필요해요"에 **"프로모션 카드 내용"** 항목 추가 + 미리보기 카드에 **코랄 형광(is-flag)**. 클릭 → 첫 카드 편집기.
- `REVIEW_FIELDS` 에 promotions 추가(단, `promoNeedsReview()` = 신뢰도 낮음 + 카드 실제 존재일 때만), `decorateEditable` 가 카드에 형광.
- 신뢰도 높으면(예: 디에이치 — PDF에서 직접 추출) 안 띄움(과잉 알림 방지). 창원(case-e)·대구범어 등 프리셋 기반은 띄움.

### (D) 개발 용어 정리
- 신뢰도를 화면엔 **친절 한국어만**(정확 / 확인 권장 / 확인 필요 / 못 찾음 + 색점) — high/medium/low/missing 노출 금지.
- "case-c · v8" 케이스 배지는 **사용자 화면에서 숨김 유지**(검증 스크립트가 읽는 테스트 훅으로만 존재, `[hidden]`).
- 검수 리스트·readout·원문 박스 어디에도 개발 용어(high/medium/case-x) 노출 0 — verify-trust 가 스캔 검사.
